# Weave EPUB Reader Privacy Notice

本说明用于概述 Weave EPUB Reader 在当前公开代码范围内涉及的数据处理行为，帮助用户理解本地数据、联网场景与许可证校验相关信息。

## 1. 适用范围

本说明适用于：

- `weave-epub-reader` 插件本体
- 当前公开仓库中可见的本地功能
- 当前公开代码中可见的许可证校验与 AI 接入行为

如果未来产品引入新的在线服务、账号体系或单独站点，以届时更新后的公开说明为准。

## 2. 本地优先原则

插件的核心阅读能力默认在本地完成，包括但不限于：

- 打开与渲染 EPUB / MOBI / AZW3 / FB2 / CBZ / TXT 等书籍文件
- 阅读进度、书签、目录导航与阅读设置
- 正文高亮、摘录、回链与来源定位
- Markdown、Canvas、Weave 数据之间的本地联动

插件不会在未触发对应在线能力时，主动上传你的 Vault 内容到外部服务。

## 3. 插件可能读取或写入的本地数据

插件可能读取或写入以下本地范围的数据：

- 当前 Obsidian Vault 中的 EPUB、Markdown、Canvas 与相关数据文件
- 插件目录中的本地状态、缓存、索引与界面配置
- 为改善使用体验而保存的本地校验缓存或授权状态

### 3.1 Vault 文件枚举

为提供书架导入、摘录回链、书签与来源定位等功能，插件会在你使用相关能力时通过 Obsidian API 枚举 Vault 中的文件路径（例如 `vault.getFiles()`、`vault.getMarkdownFiles()`），以查找支持的图书格式、Markdown / Canvas / Weave 牌组等关联内容。

插件不会因此主动上传 Vault 内容；枚举结果仅用于本地索引、匹配与界面展示。

### 3.2 剪贴板访问

在你主动触发复制操作时，插件可能读写系统剪贴板，例如：

- 复制选中的摘录或引用文本
- 复制激活码以便备份
- 复制图片（若你启用相关功能）

插件不会在后台持续读取剪贴板，也不会在你未操作时访问剪贴板。

### 3.3 本地存储

插件设置与持久化状态优先使用 Obsidian 官方插件数据 API（`loadData` / `saveData`）以及 Vault 内插件目录下的 JSON 状态文件（例如 `state/local-storage.json`）。

少量历史兼容逻辑仍可能通过 Obsidian 提供的 `App#loadLocalStorage` / `App#saveLocalStorage` 读取或清理旧键值；插件不会直接依赖浏览器 `window.localStorage` 或 `sessionStorage` 作为长期持久化方案。

## 4. 可能发生联网的场景

以下场景可能发生网络请求：

- 高级版激活
- 许可证验证
- 设备授权额度管理
- 用户自行配置的 AI 功能调用
- 用户主动接入的其他外部服务

## 5. 许可证校验可能涉及的数据

当你使用激活或许可证校验相关能力时，插件当前公开代码显示可能会发送以下信息：

- 激活码
- 购买或绑定邮箱
- 设备指纹摘要
- 平台信息

这些信息主要用于：

- 验证授权是否有效
- 管理授权设备数量
- 识别异常或重复激活
- 支持设备迁移、替换或额度控制

## 6. 设备指纹说明

当前公开代码中的设备指纹逻辑会基于设备与运行环境的一组特征生成摘要，用于授权校验与设备管理。

其目的主要是：

- 区分不同设备或运行环境
- 降低授权被滥用的风险
- 支持高级版设备额度管理

本仓库 README 中所称“设备指纹摘要”是指用于校验的摘要化结果，而不是向用户承诺某种特定的匿名化级别或法律定义。

## 7. AI 功能说明

如果你启用了 AI 相关功能，插件可能会向你配置的第三方 AI 服务发送请求。

这类请求的具体数据范围取决于：

- 你选择的 AI 服务提供商
- 你输入、选中或提交给 AI 的内容
- 你在插件中的 AI 配置

这些第三方服务不受本仓库控制，你应同时审阅相应服务商的隐私政策与使用条款。

## 8. 本地缓存

为了减少重复验证与改善体验，部分许可证验证结果可能在本地缓存一段时间。

这类缓存通常用于：

- 避免频繁重复联网验证
- 提升激活与授权状态读取速度
- 在短期内保持更稳定的使用体验

## 9. 你的数据控制权

你对自己的 Vault 内容保持控制权，包括：

- EPUB 文件
- Markdown 笔记
- Canvas 文件
- 其他位于你 Vault 内的数据

是否启用高级版、AI 能力或其他外部服务，由你自行决定。
